## Decisions / assumptions
- **No saved `/loop` or schedule artifact committed.** The plan lists that file as
  *optional*; a saved invocation would encode a concrete interval and a live session/
  routine binding that is an operator choice, not a repo artifact. The exact one-line
  invocations for both options are documented in `run.md` §2 instead. (owner: operator)
- **RESTART_INTERVAL example uses 15m**, the midpoint of the 10-20m range in
  `config.md`; the doc states the range and lets the operator pick.
- **Continuous = periodic restart, not a long-lived process.** Followed
  `agentic-operating-model.md` §4: a run drains then stops by design; the doc does not
  invent an in-script watch loop.

## Known risks / follow-ups
- The AC's live check ("add a new Todo, observe next cycle pick it up") is left as an
  operator step - it depends on a running `/loop`/schedule and live Linear, which are
  out of scope for a docs-only maker run.
- Exact `/schedule` and `/loop` argument syntax may vary slightly by harness version;
  the doc gives the intent and a working example, and both skills accept a natural-
  language body, so the invocations are robust to minor phrasing differences.

### Live (operator, when launching continuous mode)
1. Start Option A: `/loop 15m <invoke the Workflow tool with the orchestrator script>`
   (or create the Option B schedule).
2. Let one cycle drain (or run a dry-run first to see the picker output).
3. Move a fresh throwaway "add a docs line" issue in the Mini CRM project to `Todo`.
4. Observe the next cycle's picker take it and drive it Todo -> In Progress -> Done.
5. Stop by ending the `/loop` (or disabling the schedule); confirm no further runs.
