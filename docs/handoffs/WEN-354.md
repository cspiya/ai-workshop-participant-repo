detail with an "add contact to this company" flow. No schema/migration change —

- `src/app/companies/page.tsx` — list (Name, Industry, Contacts count, Website,

- `src/app/companies/new/page.tsx` — new company page.
- `src/app/companies/[id]/page.tsx` — detail (name, website, industry, notes) +

- `src/app/companies/[id]/edit/page.tsx` — edit page.
- `src/app/contacts/new/page.tsx` — reads `?companyId=` and preselects that

- `src/app/layout.tsx` — enabled the Companies nav link (was an inactive span).
- `src/lib/actions.test.ts`, `src/lib/validations.test.ts` — added company tests.

## Decisions / assumptions

- Deleting a company does NOT delete its contacts — the existing FK is
  `ON DELETE SET NULL` (verified in the F1 migration), so contacts become
  company-less. This matches the spec ("a contact may exist without a company")
  and §7.1. Delete confirmation copy states this.
- "Add contact to this company" reuses the existing contact create flow via a
  `?companyId=` query param rather than a new company-scoped action — keeps one
  create path (rule 3, keep it simple). The param is ignored if it does not match
  an existing company.
- One `companySchema` for both create and update (Company has no create-only
  defaults, unlike Contact's status), instead of separate create/update schemas.
- No spec-package exists for the Companies slice (the approved package covers
  Contacts and lists Companies as out of scope). The Linear issue is the
  executable spec for WEN-354; scope/fields taken from spec §5.1, §7.1, §8.5,
  §8.6, §11.
- `/companies` is `force-dynamic` (like `/contacts`) so newly created companies
  appear immediately and the production build needs no DATABASE_URL.

## How to verify

1. `npm run typecheck && npm run lint && npm run test && npm run build` — all exit 0.
2. `npm run db:seed` then `npm run dev`.
3. Visit `/companies` — 3 seeded companies with industry, contact counts, website.
4. New company → create with empty name (blocked) and with `website=abc` (blocked);
   create a valid one → lands on its detail.
5. Open a seeded company (e.g. Acme Logistics) → its contacts are listed; click
   "Add contact to this company" → the contact form has that company preselected;
   save → the new contact appears back on the company detail.
6. Edit the company (change industry/website) → detail reflects it.
7. Delete a company → it leaves the list; its former contacts still exist under
   `/contacts` with no company.
